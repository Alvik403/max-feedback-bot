# MAX: бот жалоб и предложений

Бот в мессенджере MAX с SQLite, веб-монитор заявок (заметки, закрытие, ответ), два режима пароля для анонимных заявок (с данными автора / без ФИО в API).

## Как назвать репозиторий

Коротко и по делу (латиница, привычно для Git):

| Название | Смысл |
|----------|--------|
| **max-feedback-bot** | Универсально, коротко |
| **complaints-suggestions-max** | Буквально пожалуйся/предложи + MAX |
| **max-ticket-desk** | «Стол заявок» + MAX |
| **corp-feedback-max** | Если внутренний/корп. контур |
| `cpz-bot` / `cpz-max-bot` | Сокращение проекта (как у вас сейчас `cpz_bot`) |

Рекомендация по умолчанию: **max-feedback-bot** или **complaints-suggestions-max** (если важно звучание «жалобы и предложения» в поиске).

## Быстрый старт (dev)

1. Скопируйте `cp .env.example .env`, заполните `MAX_BOT_TOKEN` и хотя бы один из паролей монитора.
2. `docker compose up -d --build` (или локально: `pip install -r requirements.txt`, `python bot.py` с тем же `.env`).
3. Дашборд: `http://<хост>:15000/`, проверка процесса: `GET http://<хост>:15000/health` (без авторизации).
4. В `X-Monitor-Key` передавайте пароль полного режима или режима без ФИО у анонимных (см. комментарии в `.env.example`).

## Продакшен: чеклист

- **Секреты** — только через env / secret manager; `.env` не в git (см. `.gitignore`).
- **Пароли** — разные длинные значения для `MONITOR_SECRET_FULL` (или `MONITOR_SECRET`) и `MONITOR_SECRET_REDACT`; не использовать один пароль для обоих режимов.
- **Сеть** — монитор слушает `MONITOR_HOST`/`MONITOR_PORT` (`0.0.0.0:15000` в compose); наружу только за **reverse proxy (HTTPS)** или VPN; не открывать `:15000` всему интернету без защиты.
- **Логи** — в compose задан ротационный лимит для json-file (`max-size` / `max-file`); при оркестраторе заведите свой драйвер логирования.
- **БД** — том `cpz_bot_data`; делайте **регулярные backup** файла SQLite (`/app/data/bot.sqlite` или путь из `DATABASE_PATH`).
- **Токен** MAX — как любой секрет продакшена, ротация при компрометации.
- **`docker-compose.yml`**: секция `networks.cpz_bot_net.ipam` задана из‑за исчерпания дефолтных подсетей Docker у части установок; на чистой машине её можно удалить и оставить строку `driver: bridge`.
